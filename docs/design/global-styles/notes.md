# Global Styles Design Notes

## 目的

07-global-styles で実装するCSS基盤のデザイン方針を定義する。

このメモは、`src/styles/tokens.css`、`src/styles/global.css`、`src/styles/prose.css` の判断材料として使う。完成デザインを固定するものではなく、後続のレイアウト、ナビゲーション、本文ページ、Visual Reviewで調整するための初期基準とする。

## サイトの役割

ネオン・アンダーレルムTRPGの公式ルールサイトは、長文ルール、世界観説明、キャラクター作成手順、データ参照を継続的に公開するための静的サイトである。

初期CSS基盤では、装飾よりも以下を優先する。

- 長文が読みやすい
- 見出し階層を追いやすい
- 表、箇条書き、注記、コード、引用が破綻しない
- 後続ComponentがCSS variablesを使って拡張しやすい
- GitHub Pages等の静的ホスティングで軽量に動く

## トーン

基本トーンは、読み物として落ち着いたダーク寄りのルールリファレンスとする。

ただし、全面を真っ黒にしたり、強いネオン色を広範囲に敷いたりしない。ネオン感はリンク、アクセントライン、フォーカス、コールアウトの一部に限定する。

目指す印象:

- ルールを読むための落ち着き
- 近未来・地下都市・ネオンの気配
- データ表にも耐える実務的な密度
- 装飾よりも視線誘導と可読性

避ける印象:

- ランディングページ風の派手なヒーロー
- 彩度の高いネオン色で本文が読みにくい状態
- 黒地に極細文字のサイバーパンク風UI
- 過剰なグラデーション、発光、ぼかし
- 紫・青紫だけに寄った単調な配色

## カラースキーム方針

暗めの基盤色に、青緑系とマゼンタ系のアクセントを少量使う。

想定する役割:

- page background: 低彩度の黒に近い青緑または墨色
- surface: 本文面やコードブロック用の少し明るい面
- surface muted: 表ヘッダー、引用、薄い補助背景
- text: 高コントラストだが純白ではない本文色
- text muted: 補足、メタ情報、キャプション
- border: 面を分ける低彩度の境界線
- link: 青緑系アクセント
- accent: ネオン感を出す青緑またはシアン
- danger / warning: ルール上の注意や警告に使う暖色
- focus: キーボード操作で見失わない明確な輪郭

07で決める色はCSS variables化し、後続タスクで調整しやすくする。

## タイポグラフィ方針

本文は日本語長文を優先する。

基本方針:

- system font stackを使う
- Web fontは初期スコープでは追加しない
- 本文は読みやすいline-heightを確保する
- 見出しは極端に大きくしない
- 小さな注記やメタ情報も可読性を落としすぎない
- letter-spacingは基本0とする

想定する階層:

- H1: ページタイトル
- H2: 主要セクション
- H3: ルール項目や小セクション
- body: ルール本文
- small/meta: 更新日、補足、ラベル
- code: ルート、コマンド、ID、短い識別子

## 余白・本文幅方針

ルール本文の読みやすさを優先し、本文幅を広げすぎない。

基本方針:

- prose本文は最大幅を持つ
- モバイルでは左右余白を確保する
- 見出し前の余白で階層を分かりやすくする
- 段落間は詰めすぎない
- 表やコードは横スクロールを許容し、ページ全体の横スクロールは避ける

## UI密度

運用ツールほど高密度にはしないが、ルール参照サイトとして情報量に耐える密度にする。

カードや枠を多用せず、本文、見出し、罫線、余白で構造を作る。枠はコールアウト、コード、表、個別データComponentなど、意味のある単位に限定する。

## 基本要素の方針

07では、以下のHTML要素が破綻しない状態を作る。

- `body`
- `main`
- `a`
- `h1` - `h6`
- `p`
- `ul`, `ol`, `li`
- `blockquote`
- `code`, `pre`
- `table`, `thead`, `tbody`, `tr`, `th`, `td`
- `hr`
- `strong`, `em`

高度なComponent見た目は後続タスクで扱う。

## 生成する画像

07実装前に、CSS基盤のスタイルタイル画像を生成する。

想定ファイル:

```txt
docs/design/global-styles/style-tile.png
```

必要なら、モバイル幅の確認用として以下も追加する。

```txt
docs/design/global-styles/style-tile-mobile.png
```

この画像はキービジュアルではない。07のCSS実装時に、色、文字、余白、本文密度を確認するための参照資料とする。

## 画像に含めるべき内容

`style-tile.png` には、以下が出力されているべきである。

- page background、surface、surface muted、borderの色見本
- text、text muted、link、accent、warning、danger、focusの色見本
- H1、H2、H3、本文、small/meta、inline codeの文字サンプル
- 通常本文の段落サンプル
- 箇条書きのサンプル
- 引用または補足文のサンプル
- コールアウトまたは注意枠のサンプル
- コードブロックのサンプル
- 表の簡易サンプル
- リンクの通常状態とフォーカス状態のサンプル
- PC幅での本文カラム感
- ネオン系アクセントが本文可読性を邪魔していないこと

`style-tile-mobile.png` を作る場合は、以下が分かるべきである。

- モバイル幅での本文余白
- 見出しと本文のサイズ感
- 箇条書き、コールアウト、コード、表が画面幅で破綻しないこと
- 横スクロールが必要な要素と、ページ全体が横に溢れないこと

## 07実装で使う判断基準

07-global-styles では、このメモとスタイルタイル画像をもとに、以下を実装する。

- `src/styles/tokens.css`
- `src/styles/global.css`
- `src/styles/prose.css`

実装時の判断基準:

- 色はCSS variablesとして定義する
- 本文可読性を最優先する
- ネオン表現はアクセントに限定する
- 後続Componentが使いやすい汎用トークン名にする
- ルート、コード、表、引用、コールアウトが最低限読める状態にする
- 初期スコープ外のレイアウト、ナビゲーション、検索、データComponentは実装しない

## 人間レビューで確認したい点

- ダーク寄りの方針でよいか
- ネオン感の強さが足りるか、強すぎるか
- 青緑系とマゼンタ系のアクセント配分
- 本文サイズ、見出しサイズ、行間の方向性
- 表やコードの密度
- スタイルタイル画像に追加したい表示要素
