# 生き様一覧変換仕様

## 目的

`.raw/data/ikizama-list.xlsx` を、生き様一覧ページと生き様詳細ページで共用する基礎情報のJSONへ変換する。

この仕様は、`31-0-ikizama-index-data` における現時点の変換仕様正本である。変換処理、schema、生成JSONは、この契約に従って実装する。

専用アイテムの対応表はこのExcelと生成JSONを正本として扱う。アイテム実体との整合性検証、生き様スキル、関連ページリンクなどの詳細専用データは、`32-0-ikizama-detail-data` で扱う。

## 入出力

| 項目             | 値                            |
| ---------------- | ----------------------------- |
| 入力Excel        | `.raw/data/ikizama-list.xlsx` |
| 対象シート       | `ikizama-list`                |
| 出力JSON         | `data/generated/ikizama.json` |
| JSONの`dataName` | `ikizama`                     |
| 変換コマンド     | `npm run convert:ikizama`     |
| データ取得層     | `src/lib/data/ikizama.ts`     |

Excel本体はローカル作業入力であり、Git管理しない。`data/generated/ikizama.json` は変換の出力としてGit管理し、手編集しない。CI/CDのbuildは入力Excelにも変換コマンドにも依存しない。

## 入力シート

`ikizama-list` シートは、1行目を項目グループ見出し、2行目を列見出し、3行目以降をデータ行とする。

### 1行目: 項目グループ見出し

1行目はデータへ出力しない。次のグループ見出しで列の意味を確認する。

| 列  | グループ見出し |
| --- | -------------- |
| E-F | `専用アイテム` |
| G-H | `補足`         |
| I-J | `1lv`          |
| K-L | `4lv以上`      |
| M-N | `10lv以上`     |

グループ見出し以外のセルは空欄とする。変換時は1行目の構造を検証し、列追加、列削除、グループの移動を入力エラーとする。

### 2行目: 列見出しと変換先

| 列 | Excel見出し        | JSONフィールド                                  | 入力規則                                                                           |
| -- | ------------------ | ----------------------------------------------- | ---------------------------------------------------------------------------------- |
| A  | `ID`               | `id`                                            | 必須。英小文字、数字、`-`だけを使う生き様ID。                                      |
| B  | `名称`             | `name`                                          | 必須の1行テキスト。全生き様で一意。                                                |
| C  | `短い説明`         | `shortDescription`                              | 必須の1行テキスト。一覧表示用。                                                    |
| D  | `説明`             | `description`                                   | 必須テキスト。改行を保持する。                                                     |
| E  | `専用アイテムID`   | `exclusiveItem.id`                              | 必須の1行テキスト。生き様と専用アイテム種別の対応を表すID。                        |
| F  | `専用アイテム名称` | `exclusiveItem.name`                            | 必須の1行テキスト。Eに対応する表示名称。                                           |
| G  | `補足タイプ`       | `note.type`                                     | Hと対で任意。`note`、`tip`、`warning`、`danger`、`example`、`version` のいずれか。 |
| H  | `補足本文`         | `note.content`                                  | Gと対で任意。改行を保持する。                                                      |
| I  | `体力係数`         | `secondaryAttributeCoefficients.level1.health`  | 必須の正整数。レベル1からレベル3までの係数。                                       |
| J  | `精神力係数`       | `secondaryAttributeCoefficients.level1.mind`    | 必須の正整数。レベル1からレベル3までの係数。                                       |
| K  | `体力係数`         | `secondaryAttributeCoefficients.level4.health`  | 必須の正整数。レベル4以上かつレベル10未満の係数。                                  |
| L  | `精神力係数`       | `secondaryAttributeCoefficients.level4.mind`    | 必須の正整数。レベル4以上かつレベル10未満の係数。                                  |
| M  | `体力係数`         | `secondaryAttributeCoefficients.level10.health` | 必須の正整数。レベル10以上の係数。                                                 |
| N  | `精神力係数`       | `secondaryAttributeCoefficients.level10.mind`   | 必須の正整数。レベル10以上の係数。                                                 |
| O  | `能力値ポイント`   | `attributePoints`                               | 必須。カンマ区切りの正整数4個。入力順を保持した数値配列へ変換する。                |

2行目の見出しは、上表の順序と完全一致しなければならない。同じ列見出しが複数ある係数列は、列記号と1行目のグループ見出しで区別する。

## 行の扱い

- 3行目以降を上から順に読み込む。
- 末尾の完全空行は読み込まない。
- データ領域内の完全空行、または必須列が空欄の部分行は入力エラーとする。
- 文字列の先頭・末尾の空白は除去する。説明と補足本文の改行はLFへ正規化し、内部の改行と空白は保持する。
- `id`、`name`、`shortDescription`、`note.type` は改行を許可しない。
- 係数列は文字列や小数を許可せず、正整数として読み込む。
- `能力値ポイント` は外側の空白を除去してカンマで4個に分割する。各要素は空欄、文字列、小数、0、負数を許可せず、入力順を変えない。
- `補足タイプ`と補足本文は、両方が空欄なら`note: null`、両方が入力されていれば`note`オブジェクトへ変換する。片方だけの入力はエラーとする。
- E-Fは生き様と専用アイテムの対応表であり、`exclusiveItem`としてJSONへ保持する。31-0ではIDと名称が必須の1行テキストであることだけを検証する。アイテム実体のID、個別アンカー、リンク先との整合性検証は32-0で扱う。

## IDと表示順

- 生き様IDはA列の明示値を使い、自動採番や名称からの生成をしない。
- IDは`^[a-z][a-z0-9-]*$`に一致し、全データで一意でなければならない。
- 生き様名称は全データで一意でなければならない。
- IDは後続の`/data/ikizama/[ikizamaId]`と、生き様スキルIDの所属生き様IDを結び付ける基準とする。
- 生き様の名称とA列で決定したIDの組は恒久的に不変とする。既存生き様の名称変更、ID変更、削除は認めない。
- 追加は許可する。ただし新規生き様は、既存生き様と重複しない名称とIDを持たなければならない。
- 既存の`data/generated/ikizama.json`がある再変換では、既存の全IDが出力にも残り、各IDに対応する名称が変わらないことを検証する。IDが消えた場合、既存IDに別の名称を対応させた場合、既存名称に別のIDを対応させた場合は変換エラーとする。
- 出力配列の順序はExcelの入力順とする。ページ側や取得層で独自のソートをしない。
- 各データに、Excel入力順を1から連番で記録する`sourceOrder`を追加する。重複、欠番、順序逆転を許可しない。

## 出力JSON

出力は次の形とする。`updatedAt`は変換時刻をJSTオフセット`+09:00`付きISO 8601形式で記録する。同一データを再変換した場合は、既存JSONの`updatedAt`を維持する。

```json
{
  "dataName": "ikizama",
  "updatedAt": "2026-07-22T00:00:00+09:00",
  "data": [
    {
      "id": "burai",
      "name": "ブライ",
      "shortDescription": "あえて科学技術を使わないことを選んだ生き様です。",
      "description": "あえて科学技術に頼らず生きることを選んだ人々の生き様です。",
      "exclusiveItem": {
        "id": "omamori",
        "name": "お守り"
      },
      "note": null,
      "secondaryAttributeCoefficients": {
        "level1": { "health": 11, "mind": 7 },
        "level4": { "health": 12, "mind": 9 },
        "level10": { "health": 14, "mind": 10 }
      },
      "attributePoints": [5, 4, 3, 2],
      "sourceOrder": 1
    }
  ]
}
```

このJSONは、次の表示で共用する。

- `31-2-ikizama-index-page` の生き様一覧
- `32-2-ikizama-detail-page` の生き様名、説明、補足、副能力係数、能力値ポイント、専用アイテム対応表

専用アイテムへのリンク先検証、生き様スキル一覧、関連ページリンクは、このJSONに含めない。これらは `32-0-ikizama-detail-data` で扱う。

## 検証

`IkizamaSchema`は各データ行の型、必須項目、改行、専用アイテム対応、数値、補足の組み合わせ、能力値ポイントの4要素を検証する。`IkizamaJsonSchema`または同等のassert helperは、JSON全体について次を検証する。

- `dataName`が`ikizama`であること
- `updatedAt`がJSTオフセット付きISO 8601形式であること
- `id`の書式と一意性
- `name`の一意性
- 出力配列が空でないこと
- `sourceOrder`が1からの連番で、Excel入力順と一致すること
- `exclusiveItem`のIDと名称が必須の1行テキストであること
- 各係数閾値に体力・精神力の正整数が存在すること
- `attributePoints`が正整数4個の入力順配列であること

入力エラーには、Excelの行番号、列記号、列見出しを含める。

## テスト

実Excel本体に依存しない最小fixtureを用意し、少なくとも次を確認する。

- 15列・2行ヘッダー・複数生き様を正しく変換できること
- 出力順と`sourceOrder`がExcel入力順を維持すること
- ID・名称の重複、不正なID、必須項目欠落、部分空行を拒否すること
- 既存生き様の名称変更、ID変更、削除を拒否し、新規生き様の追加を許可すること
- 補足列の片方だけの入力と、不正な補足タイプを拒否すること
- 専用アイテムID・名称の欠落を拒否し、Excelの対応表を`exclusiveItem`へそのまま変換すること
- 正整数でない係数と、4個の正整数でない能力値ポイントを拒否すること
- ヘッダー不一致、列追加、列削除を、行・列・見出しを含むエラーとして報告すること
- CRLFをLFへ正規化し、説明・補足本文の改行を保持すること
- 専用アイテムID・名称がアイテム実体、個別アンカー、リンク先と整合するかは検証しないこと
- 取得層が全生き様の入力順配列と、IDで指定した1件の生き様を返すこと
- Excelなしで、生成済み`ikizama.json`を使ったsite buildができること

## 対象外

- Excelや生成JSONを使うページUI、Component、designの実装
- 専用アイテム実体、個別アンカー、リンク先との整合性検証、生き様スキル、関連ページのデータ形状、ID形式、関連変換。これらは`32-0-ikizama-detail-data`で扱う。
- Excel上でのID自動生成、補足文の自動修正、能力値ポイントの並べ替え
- CI/CD上でのExcel変換
