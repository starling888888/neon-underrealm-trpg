# スキルデータ変換仕様

## 目的

共通スキル、流儀スキル、生き様スキルで共通するExcel入力、生成JSON、Schema、変換処理の契約を定義する。

各データ種別はこの仕様を基礎とし、入力ファイル・シート・出力先・所属データとの関連は、それぞれの
`docs/conversion/*.md` で定義する。

## 変換設定

汎用変換器は、少なくとも次の設定を受け取る。

| 設定         | 説明                                                                   |
| ------------ | ---------------------------------------------------------------------- |
| `inputPath`  | 入力Excelファイルのパス。                                              |
| `sheetName`  | 所属を持たないスキル用の単一シート名。                                 |
| `ownerKind`  | 所属を持つスキルの所属種別。`ryugi`、`ikizama`など。                   |
| `ownerIds`   | 所属を持つスキルで許可する所属IDの、表示順を持つ配列。                 |
| `outputPath` | Git管理する生成JSONの出力先。                                          |
| `dataName`   | 生成JSONの `dataName`。既存JSONを読む場合も同じ値を要求する。          |
| `idPrefix`   | 所属を持たないスキル用の固定ID prefix。共通スキルでは `skill-common`。 |

所属を持つ入力では、各シート名を所属IDとして使う。変換器は `ownerIds` とExcelシート名の集合が
完全一致することを検証し、各シートのスキルID prefixを `skill-{ownerKind}-{ownerId}` から導出する。
所属列の追加、流儀名からの所属推測、シート名以外の所属決定は行わない。

## 入力Excel

1行目をヘッダー行とし、次の12列をこの順序で受け入れる。

```txt
区分、名称、最大レベル、タイミング、コスト、技能、取得制限、対象、射程、使用制限、概要、効果
```

所属を持たないスキルは `sheetName` の単一シートを入力とする。所属を持つスキルは、`ownerIds` の
各IDと同名のシートを入力とする。どちらもスキル行へ所属ID列を追加しない。

各個別仕様で定めたヘッダー名、列数、順序が一致しない場合はエラーとする。未知ヘッダー、列の
入れ替え、`ID`列などの未定義な追加列は許可しない。

- データ行は2行目以降とする。
- 全列が空欄の行は完全空行として扱い、末尾の完全空行だけを無視する。
- データ行の途中に完全空行がある場合はエラーとする。
- 入力エラーには、修正箇所を特定できるExcel行番号、列記号、列名を含める。
- 文字列は先頭・末尾の空白を除去し、改行コードをLFへ正規化する。本文中の空白と改行は保持する。

| ヘッダー     | JSONフィールド           | 入力必須 | 仕様                                                                  |
| ------------ | ------------------------ | -------: | --------------------------------------------------------------------- |
| `区分`       | `category`               |     必須 | `bonus`、`basic`、`advanced` のいずれか。                             |
| `名称`       | `name`                   |     必須 | 空文字列はエラー。任意の文字列とLF改行を許可する。                    |
| `最大レベル` | `maxLevel`               |     必須 | 正の整数。Excel数値セル、または正の整数だけからなる文字列を許可する。 |
| `タイミング` | `timing`                 |     必須 | 単一の許容表記、または `/` で連結した許容表記の組み合わせ。           |
| `コスト`     | `cost`                   |     任意 | 1行の文字列。空欄は `null`。                                          |
| `技能`       | `proficiency`            |     任意 | 1行の文字列。空欄は `null`。                                          |
| `取得制限`   | `acquisitionRestriction` |     任意 | 1行の文字列。空欄は `null`。                                          |
| `対象`       | `target`                 |     任意 | 1行の文字列。空欄は `null`。                                          |
| `射程`       | `range`                  |     任意 | 1行の文字列。空欄は `null`。                                          |
| `使用制限`   | `usageRestriction`       |     任意 | 1行の文字列。空欄は `null`。                                          |
| `概要`       | `summary`                |     任意 | 任意の文字列とLF改行を許可する。空欄は空文字列 `""` として出力する。  |
| `効果`       | `effect`                 |     必須 | 空文字列はエラー。任意の文字列とLF改行を許可する。                    |

`概要` は、画面で表示する任意テキストであり、空欄を含めて常にstringとして扱う。

## カテゴリ、タイミング、ID

カテゴリは全スキルで `bonus`、`basic`、`advanced` の固定3区分とする。

単一タイミングは次の表記だけを受け入れる。表示用の `timing` は入力の各要素をtrimして `/` で
再結合した文字列とし、ID用の `normalizedTiming` だけを置き換える。

| タイミング | `normalizedTiming` |
| ---------- | ------------------ |
| `Pv`       | `pv`               |
| `SU`       | `su`               |
| `INI`      | `ini`              |
| `CU`       | `cu`               |
| `M`        | `m`                |
| `○-○`      | `a`                |
| `○-×`      | `a`                |
| `○-☆`      | `a`                |
| `×-○`      | `a`                |
| `×-×`      | `a`                |
| `×-☆`      | `a`                |
| `☆-○`      | `a`                |
| `☆-×`      | `a`                |
| `☆-☆`      | `a`                |
| `R`        | `r`                |
| `Aa`       | `aa`               |
| `Ra`       | `ra`               |
| `D`        | `d`                |
| `SP`       | `sp`               |

複合タイミングは、上表の単一タイミングを `/` で1個以上連結した表記とする。空の要素、未知の
要素、同じ単一タイミングの重複はエラーとする。`Aa/Ra` のような複合タイミングは許可する。

ID用の複合タイミングは、構成要素を上表の順序で正規化して `_` で連結する。入力上の順序が
異なっても、同じ組み合わせは同じ `normalizedTiming` になる。

```txt
Aa/Ra -> aa_ra
Ra/Aa -> aa_ra
○-○/R -> a_r
```

スキルIDは次の形式とする。

```txt
{idPrefix}-{category}-{normalizedTiming}-{index}
```

`index` は同じ `idPrefix`、カテゴリ、正規化タイミングのグループ内で、各入力シートのExcel
入力順に `001`から自動採番する。Excel上のID列は使わない。

`idPrefix` は、共通スキルでは `skill-common`、流儀スキルでは
`skill-ryugi-{ryugiId}`、生き様スキルでは `skill-ikizama-{ikizamaId}` とする。

## 生成JSON

所属を持たない共通スキルは、次の共通形式で出力する。

```json
{
  "dataName": "common-skills",
  "updatedAt": "2026-07-14T12:00:00+09:00",
  "data": {
    "bonus": [],
    "basic": [],
    "advanced": []
  }
}
```

所属を持つスキルは、シート名で決定した所属IDをキーにして同じカテゴリ形式を保持する。個別仕様は
所属IDの検証先と `dataName`を定義する。

```json
{
  "dataName": "example-owned-skills",
  "updatedAt": "2026-07-14T12:00:00+09:00",
  "data": {
    "owner-id": {
      "bonus": [],
      "basic": [],
      "advanced": []
    }
  }
}
```

各カテゴリ配列は、同じ所属ID・カテゴリ内のExcel入力順を保持する。`sourceOrder` は各入力シートの
2行目を1とする、同じ所属ID内で一意な正の連番とする。`Skill` は `id`、`category`、`name`、
`maxLevel`、`timing`、`cost`、`proficiency`、`acquisitionRestriction`、`target`、`range`、
`usageRestriction`、`summary`、`effect`、`sourceOrder` を持つ。所属IDを各 `Skill` に重複出力しない。

`updatedAt` はISO 8601かつJSTオフセット `+09:00` を持つ。変換器は既存JSONの `data` と
今回の `data` が同一なら既存値を維持し、異なる場合だけ今回の生成時刻へ更新する。

## タイミング順Warning

各カテゴリ内の推奨順は `A`、`R`、`Pv`、`M`、`SU`、`INI`、`CU`、`Aa`、`Ra`、`D`、`SP` とする。
`A` は9種の攻撃タイミングをまとめる内部グループラベルで、同グループ内の詳細にはWarning順を
設けない。

複合タイミングは構成要素のうち最も早いグループを順序判定に使う。優先順位が高いグループが後から
現れた場合、変換を失敗させず、シート名、カテゴリ、Excel行番号、スキル名、先行グループ、
今回のタイミング、期待順を含むWarningを標準エラー出力へ出す。

## 検証とテスト

Schemaは、固定カテゴリキー、設定した `dataName`、必須・nullable項目、ID形式・重複・
カテゴリとタイミングの整合、JSTの `updatedAt`、LF改行、`sourceOrder` の連番・一意性・
カテゴリ内昇順を検証する。

変換・Schema・取得層のテストは、入力順による採番、必須項目、無効なカテゴリ・タイミング、
複合タイミングの正規化、nullable項目、`概要`の空文字列fallback、`対象`の`null`化、
LF正規化、Warning、`updatedAt`更新規則を検証する。所属を持つスキルでは、
シート名と所属IDの集合一致、シート単位の`sourceOrder`、複数シートから1つのJSONへの集約も検証する。
