# スキルデータ変換仕様

## 目的

共通スキル、流儀スキル、生き様スキルで共通するExcel入力、生成JSON、Schema、変換処理の契約を定義する。

各データ種別は、この仕様で定義する汎用変換器へ設定を渡す。入力ファイル・シート・出力先などの個別設定は、それぞれの `docs/conversion/*.md` で定義する。

## 変換設定

汎用変換器は、次の設定を受け取る。

| 設定         | 説明                                                          |
| ------------ | ------------------------------------------------------------- |
| `inputPath`  | 入力Excelファイルのパス。                                     |
| `sheetName`  | 読み取るシート名。                                            |
| `outputPath` | Git管理する生成JSONの出力先。                                 |
| `dataName`   | 生成JSONの `dataName`。既存JSONを読む場合も同じ値を要求する。 |
| `idPrefix`   | スキルIDの `category` より前の固定部分。                      |

## 入力Excel

1行目をヘッダー行とし、次の12列をこの順序で受け入れる。

```txt
区分、名称、最大レベル、タイミング、コスト、技能、取得制限、対象、射程、使用制限、概要、効果
```

ヘッダー名、列数、順序が一致しない場合はエラーとする。未知ヘッダー、列の入れ替え、`ID` 列を含む追加列は許可しない。

- データ行は2行目以降とする。
- 全列が空欄の行は完全空行として扱い、末尾の完全空行だけを無視する。
- データ行の途中に完全空行がある場合はエラーとする。
- 入力エラーには、修正箇所を特定できるExcel行番号と列名を含める。
- すべての文字列は前後の空白を削除し、改行コードをLFへ正規化する。

| ヘッダー     | JSONフィールド           | 必須 | 仕様                                                                  |
| ------------ | ------------------------ | ---: | --------------------------------------------------------------------- |
| `区分`       | `category`               | 必須 | `bonus`、`basic`、`advanced` のいずれか。                             |
| `名称`       | `name`                   | 必須 | 1行の文字列。空欄・改行はエラー。                                     |
| `最大レベル` | `maxLevel`               | 必須 | 正の整数。Excel数値セル、または正の整数だけからなる文字列を許可する。 |
| `タイミング` | `timing`                 | 必須 | 本仕様の許容表記のいずれか。改行はエラー。                            |
| `コスト`     | `cost`                   | 任意 | 1行の文字列。空欄は `null`。                                          |
| `技能`       | `proficiency`            | 任意 | 1行の文字列。空欄は `null`。判定を伴わないスキルでは空欄にする。      |
| `取得制限`   | `acquisitionRestriction` | 任意 | 1行の文字列。空欄は `null`。                                          |
| `対象`       | `target`                 | 必須 | 1行の文字列。空欄・改行はエラー。                                     |
| `射程`       | `range`                  | 任意 | 1行の文字列。空欄は `null`。                                          |
| `使用制限`   | `usageRestriction`       | 任意 | 1行の文字列。空欄は `null`。                                          |
| `概要`       | `summary`                | 必須 | 文字列。空欄はエラー。セル内改行を許可し、LFへ正規化する。            |
| `効果`       | `effect`                 | 必須 | 文字列。空欄はエラー。セル内改行を許可し、LFへ正規化する。            |

## カテゴリ、タイミング、ID

カテゴリは全スキルで `bonus`、`basic`、`advanced` の固定3区分とする。

タイミングは次の表記だけを受け入れる。表示用の `timing` は変更せず、ID用の `normalizedTiming` だけを置き換える。

| タイミング | `normalizedTiming` |
| ---------- | ------------------ |
| `Pv`       | `pv`               |
| `SU`       | `su`               |
| `INI`      | `ini`              |
| `CU`       | `cu`               |
| `M`        | `m`                |
| `○-○`      | `a`                |
| `○-×`      | `a`                |
| `○-☆`      | `a`                |
| `×-○`      | `a`                |
| `×-×`      | `a`                |
| `×-☆`      | `a`                |
| `☆-○`      | `a`                |
| `☆-×`      | `a`                |
| `☆-☆`      | `a`                |
| `R`        | `r`                |
| `Aa`       | `aa`               |
| `Ra`       | `ra`               |
| `D`        | `d`                |
| `SP`       | `sp`               |

スキルIDは `{idPrefix}-{category}-{normalizedTiming}-{index}` とする。`index` は同じ `idPrefix`、カテゴリ、正規化タイミングのグループ内で、Excel入力順に `001` から自動採番する。

`idPrefix` は、共通スキルでは `skill-common`、流儀スキルでは `skill-ryugi-{ryugiId}`、生き様スキルでは `skill-ikizama-{ikizamaId}` とする。Excel上のID列は使わない。

## 生成JSON

生成JSONは次の共通形式とする。

```json
{
  "dataName": "example-skills",
  "updatedAt": "2026-07-14T12:00:00+09:00",
  "data": {
    "bonus": [],
    "basic": [],
    "advanced": []
  }
}
```

`data` は固定キー `bonus`、`basic`、`advanced` をすべて持つ。各カテゴリ配列はExcel入力順を保持する。`sourceOrder` はExcelの2行目を1とする全スキル共通の正の連番で、カテゴリ配列内では昇順、全カテゴリでは重複・欠番なしとする。

`Skill` は `id`、`category`、`name`、`maxLevel`、`timing`、`cost`、`proficiency`、`acquisitionRestriction`、`target`、`range`、`usageRestriction`、`summary`、`effect`、`sourceOrder` を持つ。`owner` は出力しない。

`updatedAt` はISO 8601かつJSTオフセット `+09:00` を持つ。変換器は既存JSONの `data` と今回の `data` が同一なら既存値を維持し、異なる場合だけ今回の生成時刻へ更新する。

## タイミング順Warning

各カテゴリ内の推奨順は `A`、`R`、`Pv`、`M`、`SU`、`INI`、`CU`、`Aa`、`Ra`、`D`、`SP` とする。`A` は9種の攻撃タイミングをまとめる内部グループラベルで、同グループ内の詳細にはWarning順を設けない。

優先順位が高いグループが後から現れた場合、変換を失敗させず、カテゴリ、Excel行番号、スキル名、先行グループ、今回のタイミング、期待順を含むWarningを標準エラー出力へ出す。

## 検証とテスト

Schemaは、固定カテゴリキー、設定した `dataName`、必須・nullable項目、ID形式・重複・カテゴリとタイミングの整合、JSTの `updatedAt`、LF改行、`sourceOrder` の連番・一意性・カテゴリ内昇順を検証する。

変換・Schema・取得層のテストは、入力順による採番、必須項目、無効なカテゴリ・タイミング、`null`化、LF正規化、Warning、`updatedAt` 更新規則を検証する。
