# スキル仕様

## 目的

このドキュメントは、仕事人が使うスキルがどのような情報を持つかをゲームデザイン上の仕様として定義する。

コンテンツページと `SkillCard` は、この仕様で定義する情報を表示する。画面上の見た目とComponent APIは `docs/requirements/data-display.md`、ExcelからJSONへの変換方法は将来の `docs/conversion/` 配下で定義する。

## 対象

この仕様は、共通スキル、流儀スキル、生き様スキルに共通する形式を扱う。

NPC専用スキルを追加するときは、この仕様を拡張する。

## スキルが持つ情報

各スキルは、次の情報を持つ。

| 項目       | 必須性 | 定義                                                                          |
| ---------- | ------ | ----------------------------------------------------------------------------- |
| ID         | 必須   | スキルを一意に識別するID。形式は `skill-所属種別-区分-タイミング-名称hash`。  |
| 区分       | 必須   | スキルの標準的な取得規則を表す区分。`bonus`、`basic`、`advanced` のいずれか。 |
| 名称       | 必須   | スキルを識別する名称。                                                        |
| 最大レベル | 必須   | そのスキルを取得・成長できる上限。                                            |
| タイミング | 必須   | スキルを使用するタイミング。                                                  |
| コスト     | 任意   | スキル使用時に支払うコスト。対象が自身のスキルでは空欄にできる。              |
| 技能       | 任意   | 判定を伴うスキルで使う技能。判定を伴わない場合は空欄にする。                  |
| 取得制限   | 任意   | スキルを取得するための条件。                                                  |
| 対象       | 必須   | スキル効果の対象。                                                            |
| 射程       | 任意   | スキルが届く範囲または参照する距離。対象が自身のスキルでは空欄にできる。      |
| 使用制限   | 任意   | スキルを使用できる回数・状況などの制約。制約がない場合は空欄とする。          |
| 概要       | 必須   | スキルのフレーバー的な説明。                                                  |
| 効果       | 必須   | スキルが発揮する具体的な効果。                                                |

## ID

スキルIDは、変換時に名称から自動生成する。Excel上で入力・固定しない。

形式は、次のいずれかとする。

```text
skill-common-{category}-{timing}-{nameHash}
skill-ryugi-{ryugiId}-{category}-{timing}-{nameHash}
skill-ikizama-{ikizamaId}-{category}-{timing}-{nameHash}
```

IDに使うタイミングは `docs/requirements/data-id-policy.md` のタイミング正規化表に従う。

| 要素       | IDでの表記                                                      |
| ---------- | --------------------------------------------------------------- |
| 所属       | `common`、`ryugi-{ryugiId}`、`ikizama-{ikizamaId}` のいずれか。 |
| 区分       | `bonus`、`basic`、`advanced`。                                  |
| タイミング | `docs/requirements/data-id-policy.md` の正規化値。              |
| 名称hash   | [データID管理方針](../requirements/data-id-policy.md)に従う。   |

行の途中挿入と並び替えではIDと個別アンカーを変更しない。名称、所属、区分、タイミングなど、IDに使う入力値を変更した場合のID変更は許可する。WebキャラクターシートへスキルIDを永続保存する拡張を行う段階での変更検出は、別taskで扱う。

## 区分

| 区分       | 取得規則                                                                                                                                                                                                     |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `bonus`    | 自動習得する。                                                                                                                                                                                               |
| `basic`    | 初期作成から取得できる。                                                                                                                                                                                     |
| `advanced` | 流儀レベル、生き様レベル、格のいずれかが一定値以上であることを要求する。流儀、生き様、共通スキルの各コンテンツページで、`basic` とは独立したセクションとして記載する。必要な値はそのコンテンツページで示す。 |

## タイミング

タイミングは、戦闘中にスキルを使用する時点を表す。同じキャラクターは、1つのタイミングごとに1つのスキルまたは行動しか行えない。

| 表記  | 使用する時点                                   |
| ----- | ---------------------------------------------- |
| `Pv`  | 常時効果を発揮する。                           |
| `SU`  | セットアップタイミングで使用する。             |
| `INI` | 行動決定タイミングで使用する。                 |
| `CU`  | クリンナップタイミングで使用する。             |
| `M`   | 手番の1行動として使用する。                    |
| `○-○` | コンボ中のいつでも使用する。                   |
| `○-×` | コンボの最後の攻撃でだけ使用する。             |
| `○-☆` | コンボの最後でだけ使用する。                   |
| `×-○` | コンボの最初の攻撃でだけ使用する。             |
| `×-×` | コンボの最初かつ最後の攻撃でだけ使用する。     |
| `×-☆` | コンボの最初の攻撃で使用し、コンボを終了する。 |
| `☆-○` | コンボの最初でだけ使用する。                   |
| `☆-×` | コンボの最初で使用し、最後の攻撃で終える。     |
| `☆-☆` | コンボの最初で使用し、コンボを終了する。       |
| `R`   | 攻撃に対する対応で使用する。                   |
| `Aa`  | 攻撃の判定直後に使用する。                     |
| `Ra`  | 攻撃に対する対抗判定直後に使用する。           |
| `D`   | ダメージ決定で使用する。                       |
| `SP`  | 効果本文で示された特別なタイミングで使用する。 |

`A-A` は説明用の形式であり、Excelや生成JSONで使う実表記ではない。両方の `A` には `○`、`×`、`☆` のいずれかが入り、上表の9通りだけを使用する。

## コスト

コストは、スキルを使用するために支払うリソースを表す。

| 表記     | 支払うコスト                                                                             |
| -------- | ---------------------------------------------------------------------------------------- |
| `気X`    | 気合をX点消費する。                                                                      |
| `体X`    | 体力をX点消費する。                                                                      |
| `精X`    | 精神力をX点消費する。                                                                    |
| `小銭X`  | 小銭をX点消費する。                                                                      |
| `覚悟`   | 縁1つを覚悟として使用する。その縁は失われ、そのシナリオ中に結べる縁の最大値が1減少する。 |
| `気SP`   | 支払う気合の点数に応じて効果が変わる。                                                   |
| `体SP`   | 支払う体力の点数に応じて効果が変わる。                                                   |
| `精SP`   | 支払う精神力の点数に応じて効果が変わる。                                                 |
| `小銭SP` | 支払う小銭の点数に応じて効果が変わる。                                                   |

可変コストの支払い条件は、所属側の特記事項とスキルの効果本文に従う。

複数のコストを記載する場合は、`&` で結ぶ。`&` で結ばれたコストはすべて支払う。

## 技能

技能は、判定で使用する能力を表す。

スキルの技能欄は、スキルが判定を伴うときに使う技能を示す。`Pv` など、判定を伴わないスキルでは技能を持たないため空欄にする。`能動` と `受動` は、次の技能群から選択できることを表す特殊表記である。

| 表記   | 使用できる技能                           |
| ------ | ---------------------------------------- |
| `能動` | 喧嘩、暗殺、発砲、格闘、干渉のいずれか。 |
| `受動` | 防御、回避、耐え、抵抗のいずれか。       |

複数の技能を記載する場合は、`/` で結ぶ。`/` で結ばれた技能のいずれかを使用する。

## 取得制限

取得制限は、スキルを取得するための条件を表す。

| 表記             | 条件                                                           |
| ---------------- | -------------------------------------------------------------- |
| `自動習得`       | 特定の条件で自動習得する。                                     |
| `プライマリ限定` | 所属する流儀がプライマリ流儀でなければ取得できない。           |
| スキル名         | 記載されたスキルを最大レベルで取得していなければ取得できない。 |

## 対象

対象は、スキルの効果を受ける対象を表す。

| 表記     | 対象                                                 |
| -------- | ---------------------------------------------------- |
| `自身`   | スキルを使用するキャラクター自身を対象とする。       |
| `N体`    | N体のキャラクターを対象とする。                      |
| `範囲N`  | 範囲Nを対象とする。                                  |
| `シーン` | シーンに登場している任意のキャラクターを対象とする。 |
| `特殊`   | 対象を効果本文に記載する。                           |

`N体` および `範囲N` の `N` には、数値の代わりに `LV` と記載できる。

## 射程

射程は、スキルを使用できる距離または範囲を表す。

| 表記        | 射程                                     |
| ----------- | ---------------------------------------- |
| 0以上の整数 | 自分から何マス離れているかを表す。       |
| `武器`      | 使用する武器の射程を使う。               |
| `シーン`    | シーンに登場している任意の対象を選べる。 |
| `特殊`      | 射程を効果本文に記載する。               |

## 使用制限

使用制限は、戦闘中にそのスキルを使用できる回数を表す。

| 表記   | 使用制限                       |
| ------ | ------------------------------ |
| `N/R`  | 1ラウンドにN回まで使用できる。 |
| `N/Sn` | 1シーンにN回まで使用できる。   |
| `N/Sc` | 1シナリオにN回まで使用できる。 |
| `特殊` | 使用制限を効果本文に記載する。 |

`N` には、数値の代わりに `LV` と記載できる。複数の使用制限を記載する場合は、`&` で結ぶ。

## 共通スキルの例

| 項目       | 値                                   |
| ---------- | ------------------------------------ |
| ID         | `skill-common-bonus-a-{nameHash}`    |
| 所属       | 共通スキル                           |
| 区分       | `bonus`                              |
| 名称       | 基本の一撃                           |
| 最大レベル | 1                                    |
| タイミング | `○-○`                                |
| コスト     | `気0`                                |
| 技能       | 能動                                 |
| 取得制限   | 自動習得                             |
| 対象       | 1体                                  |
| 射程       | 武器                                 |
| 使用制限   |                                      |
| 概要       | 仕事人が最初に身につける攻撃スキル。 |
| 効果       | 攻撃を行う。                         |
