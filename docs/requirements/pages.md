# ページ要件

初期公開ページ、トップページ、流儀・生き様詳細、404ページを扱う。

## 5. ページ構成

### 5.1 初期公開ページ

初期公開スコープでは、以下のページを用意する。

```text
/
/introduction
/world
/character-making
/data
/data/ryugi
/data/ryugi/[ryugiId]
/data/ikizama
/data/ikizama/[ikizamaId]
/data/common-skills
/data/items
/data/items/weapons
/data/items/armors
/data/items/omamori
/data/items/cybernetics
/data/items/nanomachines
/data/items/drugs
/rules
/rules/scenario-play
/rules/battle
/advancement
/support
/release-notes
/404
```

### 5.2 ページ詳細

#### `/`

トップページ。

タイトルロゴ、短い紹介、主要導線、最新リリースノートを表示する。クレジットは初期必須枠にしない。

#### `/introduction`

はじめに。

ゲーム概要の再掲ではなく、本作を遊ぶ前に共有する必要なもの、基本用語、ゴールデンルール、ルールサイトの読み始める導線を記載する。

ゴールデンルールは、このゲーム全体の解釈、裁定、進行の上位原則として `/introduction` を正本とする。注意枠で明確に表示する。

#### `/world`

ワールドガイド。

初期公開範囲の世界観本文、強大な敵、オオサカ副都の勢力・NPC紹介を記載する。NPC紹介は、生成JSONを参照する`NpcCard`として表示し、個別画像、二つ名、名前、セリフ、説明を含める。二つ名がある場合はルビ付きの控えめな青緑寄りの文字色で名前の直前に続け、二つ名がない場合は名前から表示する。個別画像は`id`と同名の`.webp`を使い、存在しない場合は`no_image.webp`を表示する。同じグループ内では先頭カードを左に固定して以降を左右交互にし、グループ間で順序を持ち越さない。

GM専用情報、シナリオ本文、キャンペーン本文は配置しない。

#### `/character-making`

キャラクターメイキング。

キャラクター作成の手順を記載し、必要に応じてデータ参照導線を配置する。

Webキャラクターシート、自動計算、入力フォーム、保存機能は実装しない。

#### `/data`

データトップページ。

スキルの見方、データ項目、タイミング、コスト、制限などの凡例を記載する。

スキル凡例は、専用の `SkillLegend` Componentではなく、`SkillCard` に凡例用データを渡して表示する。

アイテム凡例は、このページで全種別をまとめて表示するのではなく、各個別アイテムページ側で各Item系Cardに凡例用データを渡して表示する。

#### `/data/ryugi`

流儀一覧。対応する `.raw/contents/ryugi-index.md` のMarkdown本文とHTMLコメントを、ページ本文・可視の表示構成の正本とする。

現在のcontents指示では、heroと導入文の後に、ケンカヤの `RyugiDataSection` を用いた「流儀データの見方」と、プライマリボーナス、基礎能力値、副能力増加値、共通スキルボーナスの4項目の説明を表示する。流儀一覧は、入力順の流儀名リンクと横に配置する `shortDescription` のみを表示し、説明が長い場合は自然に折り返す。別途「流儀詳細を見る」文言や、全流儀の共通スキルボーナスを並べる表は表示しない。

#### `/data/ryugi/[ryugiId]`

各流儀の説明、基礎能力値、プライマリボーナス、共通スキルボーナス、流儀スキル一覧を表示する。

流儀スキル一覧は `CardContainer` と `SkillCard` を利用して表示する。

#### `/data/ikizama`

生き様一覧。

#### `/data/ikizama/[ikizamaId]`

各生き様の説明、生き様データ（生き様ボーナス、能力値ポイント、副能力係数）、生き様スキル一覧、専用アイテムへのリンクを表示する。

生き様スキル一覧は `CardContainer` と `SkillCard` を利用して表示する。

#### `/data/common-skills`

共通スキル一覧。

共通スキル一覧は `CardContainer` と `SkillCard` を利用して表示する。

`bonus`、`basic`、`advanced` のカテゴリを上から順に表示し、各カテゴリ内のスキルは変換済みJSONの配列順を維持する。

#### `/data/items`

アイテム説明トップ。

武器、防具、お守り、サイバネ、ナノマシン、ドラッグへの導線を配置する。

#### `/data/items/weapons`

武器リスト。

`CardContainer` と `WeaponCard` を利用して表示する。

武器凡例は `WeaponCard` に凡例用データを渡して表示する。

#### `/data/items/armors`

防具リスト。

`CardContainer` と `ArmorCard` を利用して表示する。

防具凡例は `ArmorCard` に凡例用データを渡して表示する。

#### `/data/items/omamori`

お守り一覧。

`CardContainer` と `OmamoriCard` を利用して表示する。

お守り凡例は `OmamoriCard` に凡例用データを渡して表示する。

#### `/data/items/cybernetics`

サイバネ一覧。

`CardContainer` と `CyberneticCard` を利用して表示する。

サイバネ凡例は `CyberneticCard` に凡例用データを渡して表示する。

#### `/data/items/nanomachines`

ナノマシン一覧。

`CardContainer` と `NanomachineCard` を利用して表示する。

ナノマシン凡例は `NanomachineCard` に凡例用データを渡して表示する。

#### `/data/items/drugs`

ドラッグ一覧。

`CardContainer` と `DrugCard` を利用して表示する。

ドラッグ凡例は `DrugCard` に凡例用データを渡して表示する。

#### `/rules`

ルールトップ。

基本判定、達成値、効果値、対抗判定を記載する。ゴールデンルールは `/introduction` を参照する。

#### `/rules/scenario-play`

シナリオ進行ルール。

シーン、情報収集、休息、シナリオ終了処理などを記載する。

シナリオ本文、ハンドアウト本文、キャンペーン本文は配置しない。

#### `/rules/battle`

戦闘ルール。

攻撃、リアクション、コンボ、掛け合い等を記載する。

戦闘処理支援ツール、ダイスローラー、戦闘シミュレーターは実装しない。

#### `/advancement`

キャラクター成長。

#### `/support`

サポート。

本作は多数のダイスを使うため、オンラインセッションを推奨する。オンラインセッションの準備と進め方を記載する。

特定のオンラインセッションツールを必須にせず、Webキャラクターシート、ダイスローラー、戦闘支援などの機能は実装しない。

#### `/release-notes`

更新履歴。

#### `/404`

404ページ。

## 8. トップページ要件

### FR-03. トップページ

トップページ `/` は、サイト入口、雰囲気提示、更新確認、主要導線のためのページとする。

トップページはルール本文を詳しく読ませるページにはしない。

トップページには、以下の順で要素を表示する。

1. ゲームキャッチコピー
2. タイトルロゴ
3. 最新リリースノート5件
4. ゲームのかんたんな説明

クレジットは初期必須枠にしない。必要な権利表記や協力者表記が発生した場合は、トップページ本文、将来の専用ページ、または別タスクで扱う。

### FR-03-01. ゲームキャッチコピー

ゲームの印象を伝える短いキャッチコピーを表示する。

キャッチコピーはコードに直書きせず、Markdown、MDX、Astroページ本文、または設定ファイルから編集できること。

### FR-03-02. タイトルロゴ

タイトルロゴ画像を表示する。

ロゴ画像は静的アセットとして管理し、差し替え可能にする。

ロゴ画像には適切な `alt` 属性を設定する。

ロゴ画像の表示には、必要に応じて共通画像Componentを利用できること。

### FR-03-03. 最新リリースノート

最新のリリースノートを最大5件表示する。

各リリースノート項目には、更新日、かんたんな説明を表示する。

トップページの最新リリースノートは、リリースノートJSONから自動抽出すること。

トップページ上にリリースノートを二重管理しない。

リリースノート一覧ページ `/release-notes` へのリンクを表示する。

### FR-03-04. ゲームのかんたんな説明

ゲームの概要を短く表示する。

トップページと `/introduction` でゲーム概要を重複管理しない。`/introduction` へのリンクは、ゲームを始める前の案内として表示する。世界観、キャラクター作成、基本ルールの詳細は、それぞれ該当ページへ分けて記載する。

## 11. 流儀・生き様ページ要件

### FR-05. 流儀詳細ページ

`/data/ryugi/[ryugiId]` は、流儀詳細テンプレートから生成する。

流儀ごとに個別のページファイルを作らない。

流儀詳細ページには、以下を表示する。

- 流儀名
- 概要説明
- 基礎能力値
- プライマリボーナス
- 副能力増加値
- 共通スキルボーナス
- 流儀スキル一覧

流儀スキル一覧は、Excelから変換されたスキルデータを参照して表示する。

流儀スキル一覧は `CardContainer` と `SkillCard` を利用して表示する。

流儀本文や補足説明が必要な場合は、データ側の説明フィールド、またはMarkdown / MDXの補足本文を参照できる構成にする。

### FR-06. 生き様詳細ページ

`/data/ikizama/[ikizamaId]` は、生き様詳細テンプレートから生成する。

生き様ごとに個別のページファイルを作らない。

生き様詳細ページには、以下を表示する。

- 生き様名
- 概要説明
- 任意の補足
- 生き様hero画像
- 副能力係数
- 能力値ポイント
- 生き様ボーナス
- 生き様スキル一覧
- 専用アイテム

生き様ボーナス、生き様スキル一覧、専用アイテムは、Excelから変換されたデータを参照して表示する。

生き様データ内の生き様ボーナスはH3とし、`skills.bonus` の先頭要素を `SkillCard` で表示する。desktopでは生き様ボーナス、能力値ポイント、副能力係数をこの順の1行3列で表示し、mobileでは生き様ボーナスと能力値ポイントを上段2列、副能力係数を下段の全幅で表示する。生き様スキル一覧は
`skills.basic`、`skills.advanced` をこの順に `CardContainer` と `SkillCard` で表示し、各カテゴリ内では
変換済みJSONの配列順を変えない。データがないカテゴリは見出しと一覧を表示しない。

専用アイテムは `exclusiveItem.name` をリンク文言にし、`exclusiveItem.id` に対応するアイテム一覧へリンクする。
`exclusiveItem.id` を個別アイテムIDまたは個別ItemCardのアンカーとして扱わない。

## 16. 404ページ要件

### FR-11. 404ページ

存在しないURLにアクセスした場合、404ページを表示する。

404ページは簡易な内容でよい。

最低限、以下を表示する。

- ページが見つからない旨
- トップページへのリンク
- サイトメニューまたは検索への導線

GitHub Pages等の静的ホスティングで404ページとして機能する構成にする。

404ページではページ内目次を表示しない。
