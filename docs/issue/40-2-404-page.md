# 40-2-404-page

## 目的

存在しないURLへアクセスした利用者に、画像、ページが存在しない旨、トップページへ戻るリンクを順に示すシンプルな404ページを提供する。

## 背景

`docs/plan.md` の `40-2-404-page` で、静的ホスティング上で機能する `/404` を追加することが予定されている。

対応する `.raw/contents/404.md` はローカルに存在しない。そのため、ページ本文と可視構成はユーザーの最新指示を正本とする。

- ユーザー提供の `public/images/404.webp` を先頭に表示する。
- その下に「お探しのページは存在しません」を表示する。
- さらにその下に「トップに戻る」のリンクを表示する。
- H1は文書構造のために `404 Not Found` として置くが、視覚的には非表示にする。

関連する要件と参照は以下とする。

- `docs/requirements/pages.md` の FR-11
- `docs/requirements/layout-navigation.md` の `/404` では PageToc を表示しない要件
- `docs/design/page-toc/design-desktop-hidden-404.png` と `notes.md` の、404ではPageTocの右レールを表示しない参照
- `docs/plan.md` の `40-2-404-page`

`docs/requirements/pages.md` はサイトメニューまたは検索への導線を求めるが、既存の `AppContainer` / `NoTocPageLayout` が共有SiteMenuを提供する。このissueでは、ページ本文へ追加のサイトメニューまたは検索UIを置かず、トップページへ戻るリンクだけを追加する。

## 対象範囲

- `src/pages/404.astro` を追加し、Astroの404ルートとして機能させる。
- 既存の `AppContainer` を利用し、PageTocを表示しない。
- ユーザー提供済みの `public/images/404.webp` を、既存のbase path対応方式で404ページの画像として参照する。画像バイナリはこのtaskで加工または置換しない。
- 画像の代替テキストは `404 Not Found` とする。
- 表示順を「画像」「お探しのページは存在しません」「トップに戻る」にする。
- H1を `404 Not Found` としてアクセシブルな文書構造に保持しつつ、視覚的には非表示にする。
- トップへの内部リンクでは既存のサブパス対応方式を用いる。
- 404ページをVisual Review対象に追加する。存在しないサブパスへアクセスしてdesktop・mobileのスクリーンショットだけを取得するVisual Testを追加する。

## 初期スコープ外

- 404専用の検索UI、サイトメニュー、パンくず、追加ナビゲーションを実装しない。
- PageTocおよびモバイルPageTocを表示しない。
- 実装前の404専用designドラフトを新規作成しない。
- Visual Reviewのactual screenshotを `docs/design/` へ直接コピーまたはcanonicalizeしない。
- ユーザー提供済み `public/images/404.webp` を加工、差し替え、再生成しない。
- 共通Layout、Header、Footer、SiteMenu、検索機能、PageTocの仕様を変更しない。
- 動的な404処理、SSR、CMS、分析、キャラクターシート等の初期スコープ外機能を追加しない。

## 備考

- ユーザーは「単純な画面なのでdesignドラフトスキップ」と明示した。そのため、`docs/plan.md` の「designを生成する」は実装前提から外す。実装後のVisual Reviewではactual screenshotを記録する。design正本化が必要になった場合は、ユーザー承認後に `design-image-generation` のdesign fix modeで扱う。`docs/plan.md` のチェックボックスは変更しない。
- 既存の `AppContainer` は `showPageToc` の既定値が `false` であり、`NoTocPageLayout` を選択する。実装時もこの経路を利用してPageToc非表示を保つ。
- この準備時点で `.raw/contents/404.md` は未存在である。後から作成された場合は、そのMarkdown本文とHTMLコメントをページ本文・可視構成の上位正本として再確認する。
